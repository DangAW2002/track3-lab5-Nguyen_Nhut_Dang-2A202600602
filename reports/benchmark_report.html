# GraphRAG: A Graph-Based Retrieval-Augmented Generation Architecture

## Introduction

GraphRAG (Graph-based Retrieval-Augmented Generation) is a recent approach that integrates vector search with knowledge graphs to capture both local (entity-level) and global (community-level) semantics [1]. The system is designed to handle complex questions that require synthesizing information across multiple pieces of text.

## Core Architecture

Microsoft’s implementation of GraphRAG follows a structured pipeline [1][2]:

- **Entity and relation extraction**: A large language model (LLM) extracts entities and their relationships from unstructured text, which are then organized into a structured entity-relation graph [1][2].
- **Hierarchical community detection**: The graph is partitioned into communities at multiple resolutions using the Leiden algorithm [2]. This algorithm identifies natural groupings in the data, creating a hierarchy of communities.
- **Community summary generation**: For each community, an LLM generates a natural-language summary that captures the community’s overall themes and key relations [1][2]. These summaries serve as a global index of the dataset’s topical structure.

## Query Engine

GraphRAG offers two complementary query modes [2]:

- **Local search**: Navigates the entity–relation graph and relevant text chunks to answer questions focused on specific entities.
- **Global search**: Leverages the precomputed community summaries to answer broader questions that require synthesizing information across the entire corpus.

This dual approach enables the system to answer both fact-level and thematic questions. The hierarchical community structure allows the global search to efficiently retrieve summaries at the appropriate level of abstraction, reducing the need to traverse the full graph for every query [2].
